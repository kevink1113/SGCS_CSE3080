삽입 정렬 (Insertion Sort)은
최선의 경우 (배열이 오름차순으로 정렬되어 있을 때)
배열의 원소가 이동할 필요 없이 1번씩의 비교만 수행하면 된다.
최선 시간복잡도-> O(n)

하지만 최악의 경우(배열이 내림차순으로 정렬되어 있을 때)
바깥쪽 for 루프 안의 각 반복마다 i번씩 비교해야 하므로 (n-1)+(n-2)+...+2+1 = n(n-1)/2번 비교를 하게 되고
마찬가지로 각 단계마다 i+2번씩의 이동을 하게 된다.
최악 시간복잡도->O(n^2)
결론: 내림차순 배열 상태에선 매번 비교, 이동이 일어나므로 최악의 경우이다.
